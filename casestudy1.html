Pokedex Case Study

Overview:

The JavaScript App was built to create a Pokedex full of Pokémon. It was integrated
with an already existing Pokémon database. Each Pokémon that you select you can see
the name, image, height, weight, and type.

Purpose & Context:

This was a task that was a part of my web development course. I was introduced to features
such as modals, buttons, and grids. I also was introduced to jQuery and Bootstrap. I
further advanced my experience with stylings through CSS.

Objective

The objective was to build a fully functional app that would allow users to browse through
Pokémon characters and receive their information. The aims of this project were to use a
pre-existing database and integrate it into this project.

Duration:

This project took 6-7 week for full completion. This includes a few styling additions and
issues. A few features were removed as well after trial and errors and not being necessary.

Credit:

Role: Lead Developer
Mentor: Andrew Mercer

"The How"

-An index.html file was created for our common webpage information.
-scripts.js was created to test out our Pokémon features (button) and manual information
(inserted height, weight, etc) by using functions and promises. This was to see how the
project was looking and if it was functioning properly.
-A Pokémon database was given to use. This was then implemented into the project by defining
it as apiUrl. Essentially just linking a database into the webpage we are building.

-The parameters we wanted to show off from this API were defined. In this case the name of the
Pokémon, an image, its height, weight, and class type. This was defined under the functions
loadList and loadDetails.

-After everything was implemented and working properly it just needed to spice it up a bit.
In styles.css how the buttons and the list looked were defined.

"Challenges"

The major challenges for this project were integrating a database into a project for the first
time while also trying to style the layout around it. It took some time to lose the bullet points
and structuring the page layout with grids. I also couldn’t quite get the search function to work
either. It wasn’t required and got scrapped. In the future I would like to go back and implement this.
Overall with help from my mentor we made this project fully functional.
